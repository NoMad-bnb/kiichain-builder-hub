New to KiiChain? Read This First.

Before deploying anything, understand how KiiChain works. It supports two address formats and two types of smart contracts running simultaneously on the same chain.

What is KiiChain?

Layer 1 FX Blockchain

KiiChain is a Layer 1 blockchain built on Cosmos SDK. It is designed as the first 24/7 onchain FX settlement layer for stablecoins and Real World Assets, targeting emerging markets.

Two Address Formats

EVM vs Cosmos Addresses

Your wallet has two addresses for the same account. EVM: 0x1234...abcd, Cosmos: kii1234...abcd. Both point to the same funds. Use EVM with MetaMask and Hardhat.

Two Contract Types

Solidity vs CosmWasm

Deploy Solidity contracts using Hardhat, Remix, or Foundry, just like Ethereum. Also deploy CosmWasm contracts in Rust, which are required for the RWA Protocol.

Gas Token

KII Token

KII is the native token for transaction fees. The smallest unit is akii, where 1 KII equals 10^18 akii, similar to how 1 ETH equals 10^18 wei. On Testnet, KII is free.

Current Status

Testnet Oro (Active)

KiiChain is on Testnet Oro, the permanent public testnet. Public Mainnet launch is planned for Q2 2026. All builds, hackathons, and deployments happen on Testnet Oro today.

Fee Abstraction

Pay Gas with Stablecoins

The Fee Abstraction module lets users pay transaction fees using stablecoins instead of KII. Prices are calculated via TWAP from the Oracle module, making the chain accessible to stablecoin holders.

Core Modules

5 Modules to Build With

KiiChain is purpose-built for five financial use cases. Each module has dedicated tools, contracts, and SDKs for building applications that serve emerging markets.

01 / RWA

Real World Assets

Tokenize real-world assets like real estate, bonds, or commodities using the T-REX protocol with CosmWasm contracts. Includes onchain KYC/KYB identity management, a trusted issuers registry, modular compliance rules, and ONCHAINID. Creates a unified liquidity layer for tokenized assets.

T-REX ProtocolCosmWasmKYC / KYBIdentity RegistryComplianceONCHAINID

02 / STABLECOINS

Stablecoin Layer

Create and manage non-dollar stablecoins using the TokenFactory module. The Fee Abstraction module lets users pay gas fees with stablecoins using TWAP-based Oracle pricing, so users never need to hold native KII to interact with the chain.

Gas AbstractionTokenFactoryTWAP PricingFee AbstractionERC20Oracle

03 / FX

Onchain FX Layer

The first 24/7 onchain foreign exchange settlement layer. A hybrid matching engine sources liquidity across 100+ IBC-connected chains, enabling instant cross-border payments with institutional pricing for non-dollar stablecoin pairs.

Cross-BorderIBC24/7 LiquidityInstant SettlementOracle Pricing

04 / CREDIFI

CrediFi Protocol

Decentralized credit infrastructure for businesses in emerging markets. Provides TVL-based loans, collateral management, interest rate models, and community-funded credit pools. Supports invoice financing and revenue lending for web2 and web3 companies.

TVL LoansCollateralInvoice FinancingRevenue LendingCredit Pools

05 / PAYFI

PayFi Module

Gas-optimized payment infrastructure for merchant transactions. The Paymaster system can sponsor gas so users do not need KII. Supports recurring payment scheduling, invoice financing, and TVL-based credit card financing through community Payfi pools.

PaymasterMerchant TXPayment ScheduleGas OptimizedDeFi

Deployment

4 Ways to Deploy

Choose based on your experience. Remix requires no installation and is ideal for beginners. Hardhat is recommended for serious projects. Foundry suits advanced users. CosmWasm is required for RWA contracts.

Remix IDE

Browser-based, no installation. Connect MetaMask and deploy. Start here if you are new to smart contracts.

Hardhat

JavaScript framework with scripts, testing, and env variables. Recommended for real project deployments.

Foundry

Rust-based toolkit with fast compilation. Simulate before broadcasting. Best for advanced users.

CosmWasm

Rust contracts compiled to WebAssembly. Required for the RWA Protocol T-REX implementation.

// KiiToken.sol — Save as: contracts/KiiToken.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract KiiToken {
    string public name = "KiiToken";
    string public symbol = "KII";
    uint8 public decimals = 18;
    uint256 public totalSupply;
    address public owner;
    mapping(address => uint256) public balanceOf;
    event Transfer(address indexed from, address indexed to, uint256 value);
    event Mint(address indexed to, uint256 value);

    constructor(uint256 _initialSupply) {
        owner = msg.sender;
        totalSupply = _initialSupply * (10 ** decimals);
        balanceOf[msg.sender] = totalSupply;
        emit Mint(msg.sender, totalSupply);
    }

    function transfer(address _to, uint256 _amount) public returns (bool) {
        require(balanceOf[msg.sender] >= _amount, "Not enough balance");
        require(_to != address(0), "Invalid address");
        balanceOf[msg.sender] -= _amount;
        balanceOf[_to] += _amount;
        emit Transfer(msg.sender, _to, _amount);
        return true;
    }

    function mint(address _to, uint256 _amount) public {
        require(msg.sender == owner, "Only owner can mint");
        totalSupply += _amount;
        balanceOf[_to] += _amount;
        emit Mint(_to, _amount);
    }
}

Precompiled Contracts

Access Cosmos Modules from Solidity

Precompiled contracts are built directly into KiiChain nodes. They have fixed reserved addresses where the zeros are intentional, not errors. Call them from Solidity just like any external contract. All addresses verified from docs.kiiglobal.io.

About the zero-filled addresses: Addresses like 0x0000000000000000000000000000000000000804 are real valid contract addresses on KiiChain in the reserved system address space. Call them from your Solidity code the same way you would call any other contract.

Bank

0x0000000000000000000000000000000000000804

Check token balances and total supply. Functions: balances(address), supplyOf(denom), totalSupply()

Staking

0x0000000000000000000000000000000000000800

Delegate tokens to validators, undelegate, create validators. Functions: delegate(), undelegate(), validator()

Oracle

0x0000000000000000000000000000000000001003

Read live price feeds submitted by validators. Used by Fee Abstraction for stablecoin gas pricing via TWAP.

Governance

0x0000000000000000000000000000000000000805

Vote on proposals, deposit funds, query governance state. Functions: vote(), getProposal(), getProposals()

IBC

0x0000000000000000000000000000000000001002

Send tokens cross-chain via Inter-Blockchain Communication. Functions: transfer(), transferWithDefaultTimeout()

ICS20

0x0000000000000000000000000000000000000802

Granular IBC transfer permission control over specific channels and denominations.

Wasm

0x0000000000000000000000000000000000001001

Call CosmWasm contracts from Solidity. Functions: execute(), instantiate(), querySmart(), queryRaw()

Bech32

0x0000000000000000000000000000000000000400

Convert between EVM hex (0x...) and Cosmos Bech32 (kii1...) address formats onchain.

Distribution

0x0000000000000000000000000000000000000801

Claim staking rewards, manage withdrawal addresses. Functions: claimRewards(), withdrawDelegatorRewards()

Slashing

0x0000000000000000000000000000000000000806

Query validator signing info, manage penalties. Functions: getSigningInfo(), unjail()

Vesting

0x0000000000000000000000000000000000000803

Interact with the Cosmos Vesting module for locked token and vesting schedule management.

WERC20

0xEeeeeEeeeEeEeeEeEeEeeEEEeeeeEeeeeeeeEEeE

ERC20 wrapper for native KII tokens. Enables using KII in DeFi protocols requiring standard ERC20.
